# GitHub Copilot 利用ガイド（開発者向け）

> **対象読者:** GitHub Copilot を業務で利用する開発者  
> **前提:** 本ガイドを読み、[Responsible Use 同意書](../copilot-responsible-use.html) に同意してから利用を開始してください。

---

## やることリスト（初回のみ）

1. ☐ このガイドを最後まで読む
2. ☐ [Responsible Use 同意書](../copilot-responsible-use.html) を提出する
3. ☐ 管理者から Business ライセンスの割り当てを受ける
4. ☐ エディタに Copilot 拡張をインストールする
5. ☐ 動作確認（テストリポジトリで試す）

---

## 1. ライセンスのルール

| ライセンス | 社内利用 | クライアント案件 |
|---|---|---|
| **Business**（会社契約） | ✅ 許可 | ✅ 許可 |
| Individual / Pro / Free（個人契約） | ❌ 禁止 | ❌ 禁止 |

- 業務では必ず **会社が契約した Business ライセンス** を使ってください
- 個人ライセンスには IP 補償（著作権保護）がなく、組織のポリシー管理もできないため、業務利用は禁止です
- ライセンスの申請は [管理者へ問い合わせ](../support-email.html) てください

---

## 2. 入力してはいけない情報

Copilot に以下の情報を入力・貼り付けしないでください。

| ❌ 禁止 | 具体例 |
|---|---|
| **認証情報** | API キー、パスワード、アクセストークン、秘密鍵 |
| **個人情報** | 氏名、メールアドレス、マイナンバー、電話番号 |
| **顧客データ** | 顧客の機密情報、契約情報、個人データ |
| **社外秘の独自ロジック** | 未公開のアルゴリズム、特許出願中の技術 |

> **迷ったら入力しない。** 判断に困ったら [Chat サポート](../chat-support.html) に相談してください。

---

## 3. 生成コードの扱い方

### 必ず守ること

- **レビューしてから採用する** — Copilot の提案をそのまま受け入れず、内容を確認してください
- **テストを書く** — AI 生成コードにもユニットテストを書いてください
- **コードレビューに出す** — 通常の PR フローに従ってください。AI 生成だからといって省略しないこと

### 注意点

- Copilot は**存在しない API やライブラリ**を提案することがあります（ハルシネーション）。ビルド・テストで必ず確認してください
- セキュリティ的に問題のあるコード（SQL インジェクション、ハードコードされた認証情報等）が提案されることがあります。見つけたら採用しないでください
- 提案されたコードが OSS のコピーである可能性があります。見覚えのあるコードや不自然に長い提案は注意してください

---

## 4. クライアント案件での利用

クライアント案件で Copilot を使う場合、**事前にクライアントの承認**が必要です。

### 最低限やること

1. クライアントに Copilot を使うことを伝え、承認を得る
2. 承認内容を議事録・メールに残す
3. クライアントの機密情報を Copilot に入力しない

詳細は [クライアントとの確認事項チェックリスト](../copilot-client-checklist.md) を参照してください。

## 5. データの扱い（知っておくべきこと）

| 項目 | Business ライセンスの場合 |
|---|---|
| 入力コードが AI の学習に使われる？ | **使われない** |
| データはどこで処理される？ | GitHub サーバー（デフォルト: 米国 Azure） |
| データの保持期間（IDE 補完・Chat） | **即時破棄** |
| データの保持期間（Web Chat・CLI） | **最大 28 日間** |

詳細は [データプライバシーポリシー](../copilot-data-privacy.md) を参照してください。

## 6. セットアップ手順

### VS Code の場合

1. 拡張機能で「GitHub Copilot」を検索してインストール
2. GitHub アカウントでサインイン（Business ライセンスが割り当てられたアカウント）
3. 右下のステータスバーに Copilot アイコンが表示されれば完了

### JetBrains IDE の場合

1. Settings → Plugins → Marketplace で「GitHub Copilot」を検索してインストール
2. IDE を再起動
3. GitHub アカウントでサインイン

### 動作確認

新しいファイルを開き、コメントを書いてみてください：

```python
# フィボナッチ数列を返す関数
```

グレーの文字で提案が表示されれば正常に動作しています。`Tab` で採用、`Esc` で却下です。

## 7. 便利な使い方

| 機能 | 使い方 | ショートカット（VS Code） |
|---|---|---|
| コード補完 | コメントや関数名を書くと自動提案 | `Tab` で採用 / `Esc` で却下 |
| 複数候補 | 提案を切り替えて選ぶ | `Alt + ]` / `Alt + [` |
| Copilot Chat | チャットで質問・説明・リファクタ依頼 | `Ctrl + I`（インライン） |
| テスト生成 | 関数を選択 → Chat で「テストを生成」 | — |
| ドキュメント生成 | 関数を選択 → Chat で「ドキュメントを生成」 | — |

## 8. 困ったときは

| 状況 | 対応 |
|---|---|
| Copilot が動かない | ステータスバーのアイコンを確認 → サインアウト/再サインイン |
| ライセンスエラーが出る | 管理者にライセンス割り当てを確認 → [サポートメール](../support-email.html) |
| 明らかにおかしいコードが提案される | 採用しない。必要なら [Chat サポート](../chat-support.html) に報告 |
| クライアントが AI ツール利用を禁止している | Copilot を無効化して開発してください |
| ルール違反を見つけた・してしまった | すぐに [管理者へ報告](../support-email.html)。早期報告で対応できます |

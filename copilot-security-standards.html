GitHub Copilot セキュリティ基準

GitHub Copilot を安全に利用するために守るべきセキュリティルールをまとめています。
全利用者は本基準を遵守してください。

Copilot 経由の情報漏えい・セキュリティインシデント発生時 機密情報の入力、AI生成コード起因の脆弱性等を発見した場合は、直ちに security@example.com へ連絡してください。
対象ライセンス 業務での GitHub Copilot 利用は Business / Enterprise プラン のみ許可されています。Individual / Pro / Pro+ プランでの業務利用は禁止です。
⚙

Organization 管理者設定（必須）

GitHub Organization 管理者は、Copilot を有効化する前に以下の設定を完了すること。

  • 必須ライセンスプランを Business または Enterprise に設定すること。
  • 必須Suggestions matching public code を「Blocked」に設定し、パブリックコードフィルターを有効化すること。
  • 必須Copilot 利用者を Selected teams / users に制限し、研修修了者のみに付与すること。
  • 必須機密リポジトリ・ディレクトリを Content Exclusion で Copilot の参照対象から除外すること。
  • 必須SSO（SAML / Azure AD / Okta）連携を有効にし、個人アカウントでの利用を防止すること。
  • 必須Secret Scanning を Organization 全体で有効化すること。
  • 必須Audit Log を有効化し、Copilot 関連イベントを記録すること。
  • 推奨Enterprise プランの場合、Audit Log のストリーミング（Splunk / Datadog 等）を設定すること。
🔒

入力データの保護

Copilot はIDEで開いているファイルの内容をコンテキストとしてサーバーに送信します。以下を遵守してください。

  • 禁止API キー・パスワード・トークン・秘密鍵・接続文字列をソースコード中に記述した状態で Copilot を使用すること。
  • 禁止個人情報（氏名・住所・電話番号・メールアドレス・マイナンバー等）を含むファイルで Copilot を有効にすること。
  • 禁止顧客から預かったデータ・NDA 対象の機密情報を Copilot Chat に入力すること。
  • 禁止要配慮個人情報（病歴・犯罪歴・信条・人種等）を Copilot に入力すること。
  • 必須認証情報は環境変数または GitHub Secrets / Vault 等のシークレット管理ツールで管理すること。
  • 必須.env, credentials.json 等の機密ファイルは .gitignore に追加すること。
  • 必須プレコミットフック（git-secrets 等）を導入し、機密情報パターンのコミットを自動ブロックすること。
  • 推奨機密性の高いディレクトリ（例: /secrets, /config/prod）を Content Exclusion に登録すること。
🔍

AI生成コードのセキュリティ検証

AI生成コードには脆弱性が含まれる可能性があります（約40%のケースで報告）。以下の検証を必ず実施すること。

  • 必須AI生成コードは必ず人間がレビューしてから採用すること。無レビューでの本番適用は禁止。
  • 必須SAST（静的解析）ツールによる自動スキャンを CI/CD パイプラインに組み込むこと。
  • 必須Pull Request での他者レビューを経ずに main ブランチへマージしないこと。
  • 必須AI生成コードにも通常と同等のテストカバレッジ基準を適用すること。
  • 推奨DAST（動的解析）をステージング環境で定期実行すること。
  • 推奨GitHub CodeQL を有効化し、セキュリティクエリによる自動スキャンを設定すること。

レビュー時の重点確認項目

カテゴリ 確認内容 OWASP参照
インジェクション SQL インジェクション、コマンドインジェクション、XSS の混入がないか A03:2021
認証・認可 認証バイパスや不適切な権限チェックがないか A01:2021, A07:2021
機密情報 ハードコードされた認証情報・機密値がないか A02:2021
依存関係 存在しないパッケージや悪意あるパッケージを参照していないか（タイポスクワッティング） A06:2021
暗号 非推奨の暗号アルゴリズム（MD5, SHA-1, DES等）が使われていないか A02:2021
エラー処理 過剰なエラー情報の露出（スタックトレース、内部パス等）がないか A09:2021
⚖

ライセンス・著作権リスク対策

  • 必須パブリックコードフィルター（Suggestions matching public code = Blocked）を常時有効にすること。
  • 必須SCA（Software Composition Analysis）ツールで AI生成コードの OSS ライセンス適合性を検証すること。
  • 必須GPL 等のコピーレフトライセンスに該当するコードパターンの混入に注意し、検出時は即座に対処すること。
  • 推奨Dependabot を有効化し、依存パッケージのライセンス情報を定期的に確認すること。
  • 注意IP 補償（Copyright Commitment）の適用条件を理解し、パブリックコードフィルター等の前提条件を満たすこと。
🌐

データプライバシー・データフロー

Copilot のデータ処理について理解し、適切に管理すること。

データ種別 処理内容 保持期間 学習利用
IDE コード補完 コード周辺のコンテキストをサーバーに送信 リアルタイム処理後即時削除 されない（Business/Enterprise）
Copilot Chat プロンプトをサーバーに送信 最大28日間 されない（Business/Enterprise）
提案コード サーバーからIDEに返却 キャッシュなし されない
利用メトリクス 利用率・受入率の集計 90日間 集計データのみ
  • 注意Copilot の推論処理は米国/EU のサーバーで行われる（日本リージョン未対応: 2026年4月時点）。
  • 必須個人データを Copilot に入力しないこと（越境データ移転の問題を回避）。
  • 必須金融・医療等データレジデンシー要件が厳格な案件では、プロジェクト責任者と法務が利用可否を協議すること。
  • 推奨GitHub の DPA（Data Protection Agreement）の内容を法務部が確認し、記録すること。
🔌

MCP・Extensions・Agent のセキュリティ

  • 必須MCP（Model Context Protocol）サーバーへの接続は、情報セキュリティ管理者が事前承認したもののみ許可すること。
  • 必須MCP 接続には最小権限の認証情報を使用すること（広範な PAT の使用禁止）。
  • 必須Copilot Agent（自律実行モード）の実行結果は、必ず人間がレビューしてから反映すること。
  • 必須Copilot Extensions（第三者拡張）は、情報セキュリティ管理者の事前評価・承認を経たもののみ利用可とすること。
  • 禁止未承認の MCP サーバー・Extensions を独自に接続・インストールすること。
  • 推奨Agent による特権操作（ファイル削除、デプロイ等）には human approval ゲートを設定すること。
👥

アクセス管理・ライセンス管理

  • 必須Copilot ライセンスは、ガバナンス研修テスト（15点中7点以上）に合格した者のみに付与すること。
  • 必須退職・異動時にはライセンスを速やかに回収すること（SSO 連携による自動無効化を推奨）。
  • 必須ライセンスの棚卸しを四半期ごとに実施し、未使用ライセンスを回収すること。
  • 推奨チーム単位でライセンスを管理し、プロジェクト終了時に不要ライセンスを速やかに解除すること。
  • 禁止個人の GitHub アカウント（Organization 外）で Copilot を業務利用すること。
⚠

ハルシネーション・サプライチェーン対策

Copilot は存在しない API・パッケージを提案したり、悪意あるパッケージ名に類似した名前を生成する場合があります。

  • 必須AI が提案した外部パッケージ・ライブラリの実在性を公式レジストリ（npm, PyPI 等）で確認すること。
  • 必須CI/CD パイプラインでビルド・テストを実行し、存在しない依存関係を検出すること。
  • 必須提案された API の仕様を公式ドキュメントと照合すること。
  • 推奨Dependabot / Snyk 等で依存パッケージの脆弱性・信頼性を継続監視すること。
  • 推奨パッケージのダウンロード数・メンテナンス状況を確認してから採用すること。
📊

監査・モニタリング

  • 必須Audit Log で Copilot の設定変更・ポリシー変更・ライセンス変更を記録・監視すること。
  • 必須セキュリティスキャン検出数（AI生成コード起因）を月次でモニタリングすること。
  • 必須シークレット検出数を月次で確認し、0件を維持すること。
  • 推奨利用率・提案受入率の推移を月次で確認し、異常なパターンを検知すること。
  • 推奨四半期レポートを AI-CoE に提出し、セキュリティ状況を経営層に報告すること。

定期モニタリング指標

指標 頻度 目標値 担当
セキュリティスキャン検出数（Copilot 起因） 月次 0件 セキュリティ管理者
シークレット検出・漏洩数 月次 0件 セキュリティ管理者
ライセンス棚卸し 四半期 100%完了 Org 管理者
研修修了率 四半期 100% AI-CoE
Content Exclusion 設定レビュー 四半期 全機密リポジトリ対応済み Org 管理者

🚨

インシデント対応

レベル 事象 対応時間 報告先
Critical 機密情報・個人情報の Copilot 経由での外部漏洩、本番障害 1時間以内 経営層 / セキュリティ管理者 / 法務
High AI生成コードの脆弱性が本番に混入、ライセンス違反の検出 4時間以内 AI責任者 / セキュリティ管理者
Medium テスト環境での品質問題検出、不適切なコード提案の報告 翌営業日 プロジェクト責任者
Low Copilot のサービス障害、軽微な設定不備 5営業日以内 Org 管理者
  • 必須インシデント発見者は上表に従い速やかに報告すること。
  • 必須Critical / High レベルでは該当コードの即時修正・パッチ適用を行うこと。
  • 必須全インシデントについて根本原因分析（RCA）を実施し、再発防止策を策定すること。
  • 必須個人データ漏洩が発生した場合は、個人情報保護法に基づき個人情報保護委員会に速報（3〜5日以内）・確報（30日以内）を行うこと。

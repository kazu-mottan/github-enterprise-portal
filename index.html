GitHub Enterprise

開発チームの力を、
ひとつのプラットフォームへ

GitHub Enterprise は、コードの管理・レビュー・自動化・セキュリティをエンタープライズ品質で提供する開発プラットフォームです。

はじめる

GitHub Enterprise とは？

GitHub Enterprise は、世界中の開発者が利用する GitHub の機能を、企業内のセキュリティ・コンプライアンス要件に対応した形で提供するプラットフォームです。 ソースコードの管理から、チームのコラボレーション、CI/CD パイプライン、セキュリティスキャンまで、ソフトウェア開発のライフサイクル全体をカバーします。

🔒

エンタープライズセキュリティ

SAML SSO・SCIM による ID 管理、監査ログ、IP 許可リストなど、企業要件に対応したセキュリティ機能を提供。

🤝

チームコラボレーション

Pull Request によるコードレビュー、Issue トラッキング、プロジェクト管理ボードで、チームの生産性を向上。

⚡

GitHub Actions (CI/CD)

コードのプッシュをトリガーにテスト・ビルド・デプロイを自動化。YAMLベースのワークフローで簡単に構成可能。

🛡️

Advanced Security

コードスキャン、シークレットスキャン、依存関係レビューで、脆弱性を早期に検知・修正。

主な機能

📁

リポジトリ管理

ソースコードを Git リポジトリで管理。ブランチ戦略（Git Flow / GitHub Flow）により、複数人での並行開発が可能です。アクセス権限はリポジトリ・Organization・Team 単位で細かく制御できます。

🔍

Pull Request & コードレビュー

変更内容を Pull Request（PR）として提出し、チームメンバーがレビュー・承認するワークフロー。差分の確認、コメント、提案変更が1画面で完結します。必須レビュアー設定やブランチ保護ルールで品質を担保。

🐛

Issues & プロジェクト管理

タスク・バグ・機能要望を Issue として管理。ラベル・マイルストーン・担当者を設定し、プロジェクトボード（カンバン形式）で進捗を可視化できます。

⚙️

GitHub Actions

YAML ファイルでワークフローを定義し、テスト・静的解析・ビルド・デプロイを自動実行。マーケットプレイスに公開された再利用可能なアクションを組み合わせて、素早くパイプラインを構築できます。

📦

GitHub Packages

npm・Maven・Docker などのパッケージレジストリをリポジトリと同じ場所で管理。プライベートパッケージの共有・バージョン管理が容易に行えます。

📖

Wiki & GitHub Pages

リポジトリに付属する Wiki でドキュメントを管理。GitHub Pages を使えば静的サイトをリポジトリから直接公開できます（社内イントラネット公開も設定可能）。

はじめかた

1

アカウントへのアクセス

管理者から招待メールが届きます。メール内のリンクからアカウントを有効化し、SSO（シングルサインオン）でログインしてください。

2

Git のセットアップ

ローカルPCに Git をインストールし、SSH キーまたは個人アクセストークン（PAT）を設定します。詳細は社内 Wiki の「Git セットアップガイド」を参照してください。

3

Organization / チームへの参加

所属部署の Organization に追加してもらい、必要なリポジトリへのアクセス権を取得します。チームリードまたは管理者に連絡してください。

4

最初のコントリビューション

リポジトリをクローンし、ブランチを切って変更をコミット。Pull Request を作成してレビューを依頼しましょう。詳細は「GitHub Flow ガイド」をご覧ください。

よくある質問

GitHub.com（パブリック）との違いは何ですか？

GitHub Enterprise は、企業向けに管理・セキュリティ機能が強化されたプランです。SAML SSO による認証、監査ログ、IP制限、SCIM によるユーザー管理などが追加されており、社内データを安全に管理できます。

リポジトリを誤って公開してしまった場合はどうすればよいですか？

すぐにリポジトリの設定から「Visibility」をPrivateに変更してください。その後、セキュリティチームおよびリポジトリ管理者に報告し、公開されたコンテンツに機密情報が含まれていないか確認します。

Personal Access Token (PAT) はどこで作成しますか？

GitHub にログイン後、右上のアイコン → Settings → Developer settings → Personal access tokens から作成できます。スコープは必要最小限に設定し、有効期限を設けることを推奨します。

Actions のワークフローが失敗した場合、どこを確認すればよいですか？

リポジトリの「Actions」タブから実行履歴を確認できます。失敗したジョブをクリックするとログが表示され、エラーの詳細を確認できます。

社外のメンバーをリポジトリに追加できますか？

Outside Collaborator として追加することが可能ですが、社内ポリシーの確認が必要です。追加前にセキュリティ担当者・管理者に申請してください。

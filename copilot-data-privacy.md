## ライセンス別のデータ保護レベル

| 項目 | Individual | Business（社内利用） | Enterprise |
|---|:---:|:---:|:---:|
| コードスニペットの学習利用 | あり（デフォルト） | **なし** | **なし** |
| プロンプトの学習利用 | あり（デフォルト） | **なし** | **なし** |
| 組織単位のポリシー管理 | ❌ | ✅ | ✅ |
| 監査ログ | ❌ | ✅ | ✅ |
| データ処理地域の指定 | ❌ | ❌ | ✅ |

> **社内では Business ライセンスを使用しているため、入力コードは学習データに使用されません。**

## 送信されるデータの種類

Copilot がコード補完・Chat を行う際、以下のデータが GitHub のサーバーに送信されます。

### コード補完時
- 編集中のファイルのコードスニペット（カーソル前後の文脈）
- 同じエディタで開いている関連ファイルの一部（言語・フレームワーク推定のため）
- エディタの種類・バージョン情報

### Copilot Chat 時
- チャットに入力したテキスト・貼り付けたコード
- 会話の履歴（セッション中のみ）
- 選択中のコードブロック（コンテキストとして送信）

---

## データの保持期間

| データ種別 | 保持期間 |
|---|---|
| コード補完のプロンプト・提案 | リクエスト処理後すぐに破棄 |
| Copilot Chat のメッセージ | セッション終了後に破棄（28日以内） |
| テレメトリ・使用状況ログ | 最大 28 日間（Business） |
| 監査ログ | 最大 180 日間（管理者が確認可能） |

---

## データの処理場所

- コードの処理は **GitHub のサーバー（主に米国）** で行われます。
- Enterprise ライセンスでは処理リージョンの選択が可能ですが、Business ライセンスでは指定できません。
- 社内の情報セキュリティ方針に基づき、機密性の高いコードの入力には注意してください。

---

## 学習への不使用について

Business / Enterprise ライセンスでは、GitHub の利用規約により以下が保証されています。

- 入力したコード・プロンプトは **GitHub や OpenAI の AI モデルの学習に使用されない**
- 他の組織・ユーザーへの提案にあなたのコードが使用されない

ただし、以下の点には注意が必要です。

- 上記はあくまで契約上の保証であり、**技術的に送信自体が行われないわけではありません**
- 通信はインターネット経由で行われるため、機密情報は入力しないことを推奨します

## 入力してはいけない情報

以下の情報は Copilot Chat・コード補完を問わず **絶対に入力しないでください**。

```
❌ 禁止
- API キー・アクセストークン・パスワード
- 個人情報（氏名・メールアドレス・電話番号・マイナンバーなど）
- 顧客・取引先の機密情報
- 社外秘の独自アルゴリズム・設計情報
- 契約書・法的文書の本文
- 未公開の財務情報
```

万が一入力してしまった場合は、該当のシークレット・トークンを即座にローテーションし、[セキュリティチームに報告](security-standards.html)してください。

## 管理者によるポリシー制御

Organization の管理者は以下の設定を行うことができます。

### 利用可能な制御項目

| 設定 | 内容 |
|---|---|
| Copilot の有効化・無効化 | Organization 全体または特定チームへの適用 |
| テレメトリの設定 | 使用状況データの送信 ON/OFF |
| 提案の一致表示 | パブリックコードと一致する提案の表示 ON/OFF |
| Copilot Chat の有効化 | Chat 機能の利用許可 |

### 設定場所

```
GitHub Enterprise → Organization Settings
→ Copilot → Policies
```

管理者設定の変更が必要な場合は [管理者へ問い合わせ](support-email.html) ください。
